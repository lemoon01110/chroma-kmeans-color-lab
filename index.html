How the algorithm moves

Every pixel is a point in color space. K-Means picks K random starting markers (the means), then iterates: color each point by its nearest marker, slide each marker to the average of its group, repeat until nothing budges.

1

Guess. Drop K markers at random into the cloud.

2

Assign. Every point joins its nearest marker.

3

Update. Each marker slides to its group’s average.

4

Converge. When markers stop moving, those colors are the palette.

How this works — for students

read me
Every pixel is really just three numbers — Red, Green, Blue. That makes it a single point floating in a 3-D “color space.” A whole photo is a cloud of millions of these points. K-Means’ job is to find the K best spots in that cloud.
1

Guess. Drop K markers (the “means”) into the cloud of colors.

2

Assign. Every pixel joins its nearest marker — that splits the cloud into K groups.

3

Update. Each marker slides to the average color of the pixels that joined it.

4

Converge. Repeat Assign → Update until the markers stop moving.

Then we repaint every pixel with its group’s marker color. K colors, one photo — that’s the palette on the right.

Nobody labeled the pixels or told the algorithm which colors mattered — so this is unsupervised learning. The palette is a rule the algorithm discovered by itself, not one we wrote. Hit Re-seed and you may get a slightly different answer, because it starts from a random guess.

Experiments & options
Quick experiments
Color space
How “distance” between colors is measured. LAB matches human vision, so groups often look more natural; RGB is the raw, faster option.
Where markers start
k-means++ spreads the first guesses apart (better, steadier results). random scatters them, so you’ll see more dramatic learning.
Try a new starting guess
Same photo, fresh random start — results can vary. That’s K-Means being unsupervised.
